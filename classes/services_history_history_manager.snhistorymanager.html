Class SNHistoryManager

The history manager is responsible for:

  1. Transient session history, which include keeping track of changes made in the current application session. These change logs (unless otherwise configured) are ephemeral and do not persist past application restart. Session history entries are added via change observers that trigger when an item changes.
  2. Remote server history. Entries are automatically added by the server and must be retrieved per item via an API call.
